## Quick Start

<ol class="quick-start-steps">
<li>

**Clone and generate secrets**

```bash
git clone https://github.com/fluxbase-eu/fluxbase.git
cd fluxbase/deploy
./generate-keys.sh  # Select option 1 (Docker Compose)
```

Save the **Setup Token** displayed - you'll need it for admin access.

</li>
<li>

**Start Fluxbase**

```bash
docker compose -f docker-compose.minimal.yaml up -d
```

Then open [localhost:8080/admin/setup](http://localhost:8080/admin/setup), enter your setup token, and create your admin account.

</li>
<li>

**Install the SDK and connect**

```bash
npm install @fluxbase/sdk
```

```typescript
import { createClient } from "@fluxbase/sdk";

// Get your API key from the admin dashboard ‚Üí Client Keys
const client = createClient("http://localhost:8080", "your-anon-key");

// Query data
const { data } = await client
  .from("posts")
  .select("id, title, author:users(name)")
  .eq("published", true);

// Insert data
await client.from("posts").insert({
  title: "Hello World",
  content: "My first post",
});
```

</li>
<li>

**Subscribe to real-time changes**

```typescript
client.realtime
  .channel("table:public.posts")
  .on("INSERT", (payload) => {
    console.log("New post:", payload.new_record);
  })
  .subscribe();
```

</li>
</ol>

For detailed setup instructions, see the [Quick Start Guide](/getting-started/quick-start/).

## Why Fluxbase?

<div class="comparison-highlight">

### Supabase Alternative

Fluxbase provides a similar developer experience to Supabase but runs as a **single binary** with **no external dependencies** besides PostgreSQL. Perfect for:

- **Self-hosting** on your own infrastructure
- **Air-gapped deployments** - server runs offline, clients use [REST API](/api/http/) or [vendored SDK](#air-gapped-sdk)
- **Edge deployments** where simplicity matters
- **Cost-conscious** teams who want to avoid vendor lock-in

[Compare with Supabase ‚Üí](/supabase-comparison/)

## Air-Gapped SDK {#air-gapped-sdk}

For environments without internet access, you have several options to use Fluxbase:

**Option 1: Use the REST API directly**

No SDK required - make HTTP requests to the [REST API](/api/http/) from any language.

**Option 2: Download and vendor the SDK (no npm CLI required)**

Download the package tarball directly and extract it - no npm CLI needed:

```bash
# On a machine with internet, download directly from npm registry
curl -O https://registry.npmjs.org/@fluxbase/sdk/-/sdk-1.0.0.tgz

# Transfer to your air-gapped environment, then extract
tar -xzf sdk-1.0.0.tgz
mv package node_modules/@fluxbase/sdk
```

Or use `npm pack` if available, then transfer the `.tgz` file.

**Option 3: Private registry (npm CLI optional)**

Mirror `@fluxbase/sdk` to your internal registry (Verdaccio, Nexus, Artifactory). Your package manager (npm, yarn, pnpm) just needs to be configured to point to your internal registry URL - no external internet required.

**Option 4: Use the Go client**

For Go applications, import the client directly - it compiles into your binary with no runtime dependencies:

```go
import "github.com/fluxbase-eu/fluxbase/pkg/client"
```
